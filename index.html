KERN

The Radio Astronomy Software Suite

What is KERN?

KERN is a bi-annually released set of radio astronomical software packages. It should contain most of the tools that a radio astronomer need to work with radio astronomical data.

How do I use it?

You can use our current development tree on Ubuntu 16.04: 

$ sudo apt-get install software-properties-common
$ sudo add-apt-repository -s ppa:kernsuite/kern-dev
$ sudo apt-get update
$ sudo apt-get install [package]
You can use KERN-0 (previously known as the radio-astro PPA on Ubuntu 14.04: 
$ sudo apt-get install software-properties-common
$ sudo add-apt-repository -s ppa:radio-astro/main
$ sudo apt-get update
$ sudo apt-get install [package]

What packages are available?

Have a look at the archive itself to see the list of current packages.

Why should I use KERN?

to save time and frustration setting up your scientific pipeline. Also it helps you to perform your science in a more deterministic and reproducible workflow.

What does the name KERN mean?

KERN means 'core' in Dutch and Afrikaans.

How does the release schema looks like?

We have a bleeding edge repository (kern-dev) that will be updated with the latest packages. Every half year we will freeze this repository and do a KERN release. This release will then only contain critical bug fixes but not functional changes. This all to guarantee scientific reproducibility.

What about docker images?

we have prepared easy to use base docker image which you can use to create custom docker images containers all the KERN packages combined with your own scripts.

The Dockerfile below is all you need to setup a docker container for any of the packages in kern:

FROM kernsuite/base:dev
RUN docker-apt-install aoflagger

Since it is close to impossible to make working debian packages for NRAO CASA we have created separate docker images for this software package. It is available from the docker hub:

$ docker run -ti radioastro/casa:4.6

How do I request an update or the inclusion of a package in KERN?

Open up a bug report in our packaging issue tracker. Note that we don't add packages to an existing release. Also an existing KERN release is only updated when there is a dramatically serious problem.

Why did the first KERN release get version number 0?

KERN had a presuccessor named the radio-astro launchpad PPA. This PPA is not updated anymore and renamed to KERN-0.

Why Ubuntu 16.04?

Ubuntu is one of the mostly used Linux distributions in (radio) astronomy. For now we base KERN on the latest Ubuntu LTS with a delay of 6 months up to a year. For KERN-0 this was Ubuntu 14.04, KERN-1 will be based on Ubuntu 16.04.

Can I mix KERN releases?

No, This is a really bad idea.

When is the next KERN version released?

There is no fixed release date, but we intent to release a KERN release every 6 months.
